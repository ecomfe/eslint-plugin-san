# san/attributes-order

enforce order of attributes

  • ‚öôÔ∏è This rule is included in "plugin:san/recommended".
  • üîß The --fix option on the command line (opens new window) can automatically fix some of the problems reported by this rule.

# üìñ Rule Details

This rule aims to enforce ordering of component attributes. The default order is:

  • LIST_RENDERING e.g. 's-for item in items'
  • CONDITIONALS e.g. 's-if', 's-else-if', 's-else'
  • GLOBAL e.g. 'id'
  • UNIQUE e.g. 'ref', 'key', 's-slot', 'slot'
  • OTHER_ATTR e.g. 'custom-prop="foo"', 's-bind={}', 'prop=""'
  • EVENTS e.g. 's-on="event"'
